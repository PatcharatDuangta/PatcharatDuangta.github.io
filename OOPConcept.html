ข้อ I: โค้ดตัวอย่างใน Java, C++, และ Python

Java Code:

Java 

class Animal {
    Animal() {
        System.out.println("create animal");
    }

    protected void finalize() {
        System.out.println("delete animal");
    }

    void sound() {
        System.out.println("animal makes a sound");
    }
}

class Dog extends Animal {
    Dog() {
        System.out.println("dog");
    }

    @Override
    protected void finalize() {
        System.out.println("delete dog");
    }

    @Override
    void sound() {
        System.out.println("dog barks");
    }
}

class Cat extends Animal {
    Cat() {
        System.out.println("cat");
    }

    @Override
    protected void finalize() {
        System.out.println("delete cat");
    }

    @Override
    void sound() {
        System.out.println("cat meows");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a = new Animal();
        Dog d = new Dog();
        Cat c = new Cat();

        a.sound();
        d.sound();
        c.sound();

        a = null;
        d = null;
        c = null;
        System.gc();
    }
}

C++ Code:

C++ 

#include <iostream>
using namespace std;

class Animal {
public:
    Animal() {
        cout << "create animal" << endl;
    }
    
    virtual ~Animal() {
        cout << "delete animal" << endl;
    }

    virtual void sound() {
        cout << "animal makes a sound" << endl;
    }
};

class Dog : public Animal {
public:
    Dog() {
        cout << "dog" << endl;
    }

    ~Dog() {
        cout << "delete dog" << endl;
    }

    void sound() override {
        cout << "dog barks" << endl;
    }
};

class Cat : public Animal {
public:
    Cat() {
        cout << "cat" << endl;
    }

    ~Cat() {
        cout << "delete cat" << endl;
    }

    void sound() override {
        cout << "cat meows" << endl;
    }
};

int main() {
    Animal* a = new Animal();
    Animal* d = new Dog();
    Animal* c = new Cat();

    a->sound();
    d->sound();
    c->sound();

    delete a;
    delete d;
    delete c;

    return 0;
}

ข้อ II: การทำความเข้าใจส่วนประกอบของโค้ด

1. Class (คลาส)

คลาสคือแบบพิมพ์ (blueprint) สำหรับการสร้างออบเจ็กต์ ซึ่งเป็นการรวบรวมข้อมูล (attributes) และพฤติกรรม (methods) ไว้ในรูปแบบที่สามารถนำไปใช้สร้างออบเจ็กต์ใหม่ได้ ตัวอย่างในโค้ดคือ:

class Animal

คลาส Animal ในโค้ดเป็นแบบพิมพ์สำหรับสัตว์ทั่วไป ที่มีเมธอด sound เพื่อแสดงพฤติกรรมในการส่งเสียง

ข้อ II: การทำความเข้าใจส่วนประกอบของโค้ด

1. Class (คลาส)

คลาสคือแบบพิมพ์ (blueprint) สำหรับการสร้างออบเจ็กต์ ซึ่งเป็นการรวบรวมข้อมูล (attributes) และพฤติกรรม (methods) ไว้ในรูปแบบที่สามารถนำไปใช้สร้างออบเจ็กต์ใหม่ได้ ตัวอย่างในโค้ดคือ:

class Animal

คลาส Animal ในโค้ดเป็นแบบพิมพ์สำหรับสัตว์ทั่วไป ที่มีเมธอด sound เพื่อแสดงพฤติกรรมในการส่งเสียง

2. Constructor (เมธอดสร้าง)

เมธอดสร้าง (Constructor) เป็นฟังก์ชันพิเศษที่ถูกเรียกเมื่อมีการสร้างออบเจ็กต์ใหม่ ตัวอย่างเช่น:

Animal() {
    System.out.println("create animal");
}

ในตัวอย่างนี้ เมื่อมีการสร้างออบเจ็กต์จากคลาส Animal เมธอดสร้างจะถูกเรียกและแสดงข้อความ "create animal"

3. Destructor (เมธอดทำลาย)

ในภาษาเช่น C++ เมธอดทำลาย (Destructor) จะถูกเรียกเมื่อออบเจ็กต์ถูกลบออกจากหน่วยความจำ แต่ใน Java เมธอด finalize() จะถูกใช้แทนในบางกรณี:

protected void finalize() {
    System.out.println("delete animal");
}

เมธอดนี้จะถูกเรียกโดย System.gc() ซึ่งเป็นการขอให้ Garbage Collector ลบออบเจ็กต์ที่ไม่ได้ใช้อีกต่อไป

4. Inheritance (การสืบทอด)

การสืบทอด (Inheritance) คือการสร้างคลาสใหม่จากคลาสเดิม โดยคลาสลูกจะสามารถใช้หรือปรับปรุงพฤติกรรมของคลาสพ่อแม่ได้ ในตัวอย่างนี้:

class Dog extends Animal {
    Dog() {
        System.out.println("dog");
    }

    @Override
    void sound() {
        System.out.println("dog barks");
    }
}

คลาส Dog สืบทอดมาจาก Animal และมีการ Override เมธอด sound() เพื่อเปลี่ยนพฤติกรรมให้ส่งเสียงเห่าแทน

5. Polymorphism (พหุสัณฐาน)

Polymorphism หมายถึงความสามารถในการใช้เมธอดเดียวกันในคลาสที่ต่างกัน โดยที่มีพฤติกรรมต่างกันได้ ตัวอย่างคือ:

Animal a = new Dog();
a.sound(); // Output: dog barks

แม้ว่า a จะเป็นตัวแปรชนิด Animal แต่เนื่องจากมันเก็บออบเจ็กต์ชนิด Dog เมธอด sound() ที่ถูกเรียกจึงเป็นของ Dog ซึ่งจะพิมพ์ "dog barks"

6. Overriding (การเขียนทับ)

การเขียนทับ (Overriding) หมายถึงการที่คลาสลูกปรับเปลี่ยนการทำงานของเมธอดที่สืบทอดมาจากคลาสพ่อแม่ เช่น:

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("dog barks");
    }
}

เมธอด sound() ของคลาส Dog ถูกเขียนทับจากคลาส Animal เพื่อให้แสดงผลลัพธ์ที่แตกต่างกัน

ข้อ III: Abstract Class

ในภาษา OOP (Object-Oriented Programming) Abstract Class คือคลาสที่ไม่สามารถสร้างออบเจ็กต์โดยตรงจากมันได้ แต่ถูกออกแบบมาเพื่อเป็นแบบพิมพ์ให้กับคลาสที่สืบทอดต่อ (subclass) โดยทั่วไปแล้ว abstract class จะมีเมธอดที่ยังไม่ได้ถูกกำหนดรายละเอียดไว้ (abstract methods) เพื่อให้ซับคลาสทำการกำหนดพฤติกรรมเฉพาะของตัวเอง

หลักการสร้าง Abstract Class

การสร้าง abstract class จะทำโดยใช้คีย์เวิร์ด abstract ในบางภาษา เช่น Java และ C++ ใน Python จะใช้เมตาคราส (metaclass) ซึ่งคลาส abstract จะมีทั้งเมธอดปกติและเมธอด abstract ที่ไม่กำหนดพฤติกรรม

ตัวอย่าง Abstract Class ในข้อ I

ในตัวอย่างข้อ I ของคุณ ไม่มีการใช้ abstract class แต่เราสามารถสร้าง abstract class ได้หากต้องการให้คลาส Animal เป็นคลาสแม่ที่บังคับให้คลาสลูก เช่น Dog และ Cat ต้องกำหนดพฤติกรรมของเมธอด sound() เอง โดยไม่สามารถใช้เมธอดจาก Animal ได้โดยตรง

ตัวอย่างการสร้าง Abstract Class ใน Java

abstract class Animal {
Animal() {
System.out.println("create animal");
}
abstract void sound(); // เมธอด abstract ไม่มีการกำหนดพฤติกรรม
}

class Dog extends Animal {
Dog() {
System.out.println("dog");
}
void sound() { // กำหนดพฤติกรรมสำหรับ Dog
System.out.println("dog barks");
}
}

class Cat extends Animal {
Cat() {
System.out.println("cat");
}
void sound() { // กำหนดพฤติกรรมสำหรับ Cat
System.out.println("cat meows");
}
}

ในโค้ดนี้ Animal เป็น abstract class ซึ่งมีเมธอด abstract sound() ที่ไม่มีการกำหนดพฤติกรรม คลาสลูก Dog และ Cat ต้องกำหนดพฤติกรรมของ sound() เอง

Abstract Class ในภาษาอื่น ๆ

ใน C++ การสร้าง abstract class จะทำได้โดยการกำหนดเมธอดบริสุทธิ์ (pure virtual function) ซึ่งคลาสลูกต้องทำการกำหนดพฤติกรรมของเมธอดนั้นเอง ส่วนใน Python การสร้าง abstract class จะใช้ abc (Abstract Base Class) ที่มาพร้อมกับไลบรารีมาตรฐาน

สรุป

Abstract Class ใช้เมื่อเราต้องการบังคับให้ซับคลาสกำหนดพฤติกรรมที่เฉพาะเจาะจงเอง โดยไม่ให้สามารถสร้างออบเจ็กต์จากคลาส abstract โดยตรงได้
